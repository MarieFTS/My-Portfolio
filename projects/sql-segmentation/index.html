SQL Customer Segmentation Analysis

Led a team project analyzing retail customer data using RFM (Recency, Frequency, Monetary) analysis to identify 7 distinct customer segments and drive targeted marketing strategies.

Jan 2023 – May 2023 Team Project MySQL 8 Queries

Overview

This team project focused on leveraging SQL and advanced data analysis techniques to understand customer behavior patterns. Using RFM analysis—a proven retail analytics methodology—we segmented customers into 7 distinct groups, enabling data-driven marketing strategies that resulted in a 40% revenue concentration in our Champions segment.

Process & Methodology

1

Database Design

Created customers and orders tables with proper foreign keys and indexing for optimal query performance.

2

RFM Calculation

Computed recency days, frequency, and monetary value per customer using date functions and aggregations.

3

RFM Scoring

Scored each metric 1–5 using CASE statements and percentile ranking across the customer base.

4

Segmentation

Mapped RFM combinations to 7 segments: Champions, Loyal, At Risk, Can't Lose, New, Promising, and Lost.

5

Demographics Breakdown

Analyzed age groups, gender, and geography distribution within each segment.

6

Category Preferences

Identified which product categories each segment prefers for targeted cross-selling.

RFM Scoring Implementation

SQL 
-- RFM Scoring View
CREATE OR REPLACE VIEW rfm_scores AS
SELECT *,
    CASE
        WHEN recency_days <= 30  THEN 5
        WHEN recency_days <= 60  THEN 4
        WHEN recency_days <= 120 THEN 3
        WHEN recency_days <= 180 THEN 2
        ELSE 1
    END AS recency_score,
    CASE
        WHEN frequency >= 20 THEN 5
        WHEN frequency >= 12 THEN 4
        WHEN frequency >= 6  THEN 3
        WHEN frequency >= 2  THEN 2
        ELSE 1
    END AS frequency_score,
    CASE
        WHEN monetary_value >= 5000 THEN 5
        WHEN monetary_value >= 2000 THEN 4
        WHEN monetary_value >= 800  THEN 3
        WHEN monetary_value >= 200  THEN 2
        ELSE 1
    END AS monetary_score
FROM rfm_raw;
