IT Support Analytics Dashboard

Built an IT support analytics system at CyberMunk to track ticket volume, resolution times, SLA compliance, and agent performance. Contributed to a 20% reduction in ticket resolution time through data-driven process improvements and FAQ-style documentation.

May 2024 – Present CyberMunk MySQL + Power BI Live Dashboard

Overview

This ongoing project at CyberMunk demonstrates how structured data analysis can drive operational improvements in IT support. By building comprehensive SQL views to track ticket metrics, SLA compliance, and agent performance, we identified process bottlenecks and patterns. The resulting dashboards in Power BI enabled management to make data-driven staffing decisions and prioritize FAQ documentation for the highest-impact issues, achieving a 20% reduction in resolution time.

Process & Methodology

1

Schema Design

Created tickets and agents tables with ENUM priority/status fields and proper indexing.

2

Core Metrics View

Computed resolution time in minutes/hours and SLA compliance flags per priority level.

3

Resolution Analysis

Averaged resolution time by category and priority with SLA compliance percentage.

4

Monthly Trends

Used LAG() window function to track month-over-month resolution time improvements.

5

Peak Volume Analysis

Identified busiest hours and days to optimize staffing schedules proactively.

6

FAQ Documentation

Extracted top recurring categories to guide knowledge base and reduce repeat tickets.

Monthly Trend Analysis with Window Functions

SQL 
-- Monthly trend with LAG() for MoM improvement
SELECT
    month,
    COUNT(*) AS total_tickets,
    ROUND(AVG(resolution_hours), 2) AS avg_resolution_hrs,
    ROUND(AVG(resolution_hours)
        - LAG(AVG(resolution_hours))
          OVER (ORDER BY month), 2) AS mom_change_hrs
FROM ticket_metrics
GROUP BY month
ORDER BY month;
